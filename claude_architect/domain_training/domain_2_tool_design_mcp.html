Tool Design & MCP Integration

Exam Weight: 18%

Tools are how Claude interacts with the world. A Claude agent without well-designed tools is like a brilliant engineer with no hands — it can reason perfectly but cannot act effectively.

The exam will present broken tool setups — misrouted calls, ambiguous descriptions, unhelpful error responses — and ask you to diagnose and fix them.

Bad vs. Good Tool Description

Bad — Causes Misrouting

"name": "analyze_content"
"description": "Analyzes content and returns results."

Good — Precise Routing

"name": "extract_web_results"
"description": "Analyzes raw web search result HTML/JSON to extract titles, URLs, dates, and excerpts. Use ONLY for web search output — not for PDF documents or database records. Returns list of {title, url, date, excerpt}. If input is not web search format, returns error."

Splitting Generic Tools

# BEFORE: One generic tool (Claude guesses)
analyze_document(document, instruction="do whatever seems right")

# AFTER: Three purpose-specific tools
extract_data_points(document)        # structured facts, numbers, entities
summarize_content(document)          # narrative summary
verify_claim_against_source(document, claim)  # check if claim is supported
Overlapping Tool Names — Misrouting Trap

analyze_content and analyze_document with similar descriptions will cause misrouting. Rename to reflect actual purpose: extract_web_results and analyze_pdf_document.

2.2

Structured Error Responses for MCP Tools

The Four Error Categories

transient
Timeout, service unavailable, rate limit
→ Retry with backoff
validation
Invalid input format, missing required field
→ Fix the input and retry
permission
Auth failure, insufficient access
→ Do not retry; escalate
business
Policy violation, threshold exceeded
→ Do not retry; follow business logic

The isRetryable Flag — Most Important Error Field

# Transient — retry may succeed
{
    "isError": True,
    "errorCategory": "transient",
    "isRetryable": True,
    "retryAfterSeconds": 5,
    "message": "Search service temporarily unavailable."
}

# Business rule — retrying will NOT help
{
    "isError": True,
    "errorCategory": "business",
    "isRetryable": False,
    "message": "Refund $750 exceeds automated limit of $500.",
    "requiredAction": "Call escalate_to_human with refund details."
}

Empty Results vs. Access Failures

Critical Distinction

Returning {"results": []} for BOTH empty queries AND database failures makes them indistinguishable to the coordinator. Always use a proper error structure for failures.

2.3

Distribute Tools Across Agents

Principle of Least Tool Access

More tools = worse tool selection. Agents with specialized toolsets outperform generalist agents. Give each agent the minimum tools required for its role.

# WRONG: Give all agents all tools
coordinator_tools = all_18_tools

# RIGHT: Scoped toolsets
coordinator_tools = ["Task", "summarize", "evaluate_coverage"]
search_agent_tools = ["web_search", "fetch_url", "filter_results"]
synthesis_agent_tools = ["verify_fact", "compile_report", "format_output"]

MCP Server Integration

Config LocationScopeUse Case
.mcp.json (project root)All team members who cloneShared team tooling: GitHub, Jira, internal APIs
~/.claude.json (user home)Only that developerPersonal tools, experimental servers, dev credentials

Configuring .mcp.json

{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": { "GITHUB_TOKEN": "${GITHUB_TOKEN}" }
    },
    "jira": {
      "command": "node",
      "args": ["./tools/jira-mcp-server.js"],
      "env": { "JIRA_API_TOKEN": "${JIRA_API_TOKEN}" }
    }
  }
}
Key Rule

Team tooling goes in .mcp.json (version controlled). Personal/experimental servers go in ~/.claude.json (never committed). Always use ${VARIABLE} expansion — never hardcode credentials.

MCP Resources

Resources pre-expose content catalogs, reducing exploratory tool calls. Instead of Claude calling list_projects() → list_repos() → list_issues(), an MCP resource exposes the catalog directly so Claude reads the specific item immediately.

Grep = searching inside files (content)  |  Glob = searching for files (names/paths)

What the Exam Will Test You On

"Tool calls are being misrouted between two tools with similar names. Fix it."
Rename tools and rewrite descriptions to eliminate overlap
"A subagent returns {results: []} for both empty queries AND access failures. Why is this a problem?"
Coordinator cannot distinguish "no data" from "query failed" — cannot make recovery decisions
"You need guaranteed structured output from Claude. Which tool_choice setting?"
"any" (must call a tool) or forced (must call specific tool)
"Where do you configure a shared GitHub MCP server for your team?"
.mcp.json in the project root with ${GITHUB_TOKEN} env expansion
"Claude keeps using Grep to find test files instead of Glob. What's wrong?"
Glob is for path/name matching; Grep is for content. Wrong tool for the task.
