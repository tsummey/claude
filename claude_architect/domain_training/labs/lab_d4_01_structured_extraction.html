Lab D4-01

Structured Extraction & Prompt Engineering

Domain 4 — Prompt Engineering Task Statements 4.1, 4.2, 4.3 ⏱ 45 minutes

Objective: Extract precision values from unstructured customer messages into typed Python objects. Measure extraction accuracy against a labeled dataset. Build the "case facts block" pattern to protect critical values from context window summarization. Domain 4 is 20% of the exam.

Key Insight: Agents fail not because Claude can't reason — they fail because critical facts (order IDs, amounts, dates) get paraphrased or lost in context compression. The case facts block pins exact values outside the summarized history.

Part 1

Extraction Schema Design (15 min)

from dataclasses import dataclass
from typing import Optional
import json, anthropic

@dataclass
class CaseFacts:
    # Exact precision values extracted from customer message
    customer_id: Optional[str] = None        # e.g., "C001"
    order_id: Optional[str] = None           # e.g., "ORD-5001"
    refund_amount: Optional[float] = None    # e.g., 189.99
    issue_type: Optional[str] = None         # "damaged", "wrong_item", "not_received"
    human_requested: bool = False            # Explicit request for human agent

EXTRACTION_PROMPT = """Extract case facts from this customer message.
Return ONLY valid JSON with these exact keys:
- customer_id: string or null
- order_id: string or null (format: ORD-XXXX)
- refund_amount: number or null (USD)
- issue_type: one of ["damaged", "wrong_item", "not_received", "other"] or null
- human_requested: boolean (true if customer explicitly asks for human/agent/person)

Customer message:
{message}"""

client = anthropic.Anthropic()

def extract_case_facts(message: str) -> CaseFacts:
    response = client.messages.create(
        model="claude-opus-4-5",
        max_tokens=256,
        messages=[{"role": "user", "content": EXTRACTION_PROMPT.format(message=message)}]
    )
    text = response.content[0].text.strip()
    if text.startswith("```"):
        text = text.split("\n", 1)[1].rsplit("```", 1)[0]
    data = json.loads(text)
    return CaseFacts(**data)
Part 2

Labeled Dataset & Accuracy Measurement (15 min)

TEST_CASES = [
    {
        "message": "Hi, I'm customer C001 and my order ORD-5001 arrived completely smashed. I want my $189.99 back.",
        "expected": CaseFacts(customer_id="C001", order_id="ORD-5001", refund_amount=189.99, issue_type="damaged", human_requested=False)
    },
    {
        "message": "I received the completely wrong product! Order number ORD-5002. I need to speak with a real person.",
        "expected": CaseFacts(customer_id=None, order_id="ORD-5002", refund_amount=None, issue_type="wrong_item", human_requested=True)
    },
    {
        "message": "My package never arrived. I ordered it 3 weeks ago. Customer ID: C002, order ORD-5003.",
        "expected": CaseFacts(customer_id="C002", order_id="ORD-5003", refund_amount=None, issue_type="not_received", human_requested=False)
    },
    {
        "message": "I want a full refund of seven hundred and forty-nine dollars for my damaged order ORD-5002.",
        "expected": CaseFacts(customer_id=None, order_id="ORD-5002", refund_amount=749.0, issue_type="damaged", human_requested=False)
    },
]

def measure_accuracy() -> float:
    correct = 0
    for i, case in enumerate(TEST_CASES):
        extracted = extract_case_facts(case["message"])
        expected = case["expected"]
        fields = ["customer_id", "order_id", "refund_amount", "issue_type", "human_requested"]
        field_correct = sum(1 for f in fields if getattr(extracted, f) == getattr(expected, f))
        accuracy = field_correct / len(fields)
        correct += accuracy
        print(f"Case {i+1}: {field_correct}/{len(fields)} fields correct ({accuracy:.0%})")
        if accuracy < 1.0:
            for f in fields:
                got, exp = getattr(extracted, f), getattr(expected, f)
                if got != exp:
                    print(f"  ❌ {f}: got={got!r}, expected={exp!r}")
    overall = correct / len(TEST_CASES)
    print(f"\nOverall accuracy: {overall:.1%} (target: 95%+)")
    return overall

measure_accuracy()
Case 4 Gotcha — Written-Out Numbers

"seven hundred and forty-nine dollars" — does Claude extract this as 749.0? This is a common extraction challenge. If it fails, add a few-shot example showing the numeric conversion to the extraction prompt.

Part 3

The Case Facts Block Pattern (15 min)

When conversation history gets long, Claude compresses it. Precision values get paraphrased. "Order ORD-5001 for $189.99" becomes "a damaged order" — losing the exact values you need to call process_refund. The case facts block solves this.

❌ Without Case Facts Block
# Claude sees summarized history
system = """You are a support agent..."""

messages = [
    {"role": "user", "content": "[Long conversation history here... customer had an issue with an order and wanted a refund...]"},
]
# Critical: exact order ID, amount, customer ID
# are now buried in summarized prose
✅ With Case Facts Block
# Case facts pinned in system prompt
system = """You are a support agent.

<case_facts>
customer_id: C001
order_id: ORD-5001
refund_amount: 189.99
issue_type: damaged
human_requested: false
</case_facts>

These values are authoritative. Never infer
different values from conversation history."""

messages = [
    {"role": "user", "content": "[Summarized history...]"},
]
def build_system_prompt_with_facts(facts: CaseFacts) -> str:
    facts_xml = f"""
<case_facts>
customer_id: {facts.customer_id or 'unknown'}
order_id: {facts.order_id or 'unknown'}
refund_amount: {facts.refund_amount or 'not_stated'}
issue_type: {facts.issue_type or 'unknown'}
human_requested: {str(facts.human_requested).lower()}
</case_facts>

These case facts are authoritative. Use exact values from <case_facts> when
calling tools. Never re-derive amounts, order IDs, or customer IDs from
summarized conversation history.
"""

Why XML Tags?

XML-style tags give Claude unambiguous structure markers that survive context summarization. Claude is trained to respect content inside tags as a named block. <case_facts> tells Claude "this is the authoritative fact block, not conversational history."

Q

Reflection Questions

1. In Case 4, did Claude correctly extract 749.0 from "seven hundred and forty-nine dollars"? If not, how did you fix the prompt?
Adding a few-shot example in the extraction prompt with a written-out number usually fixes this.
2. Why is the case facts block placed in the system prompt rather than in the conversation history?
System prompt is not summarized. Conversation history is. Where should authoritative facts live?
3. A customer says: "I want all of my money back." What is the correct extracted refund_amount?
Null — "all of my money" is not a specific amount. The agent must look up the order total.
4. What happens to extraction accuracy if you use Claude to summarize the conversation and then try to extract facts from the summary?
Summaries paraphrase. "ORD-5001" might become "a recent order." Always extract from the raw original message.
5. You run the accuracy test and get 80%. What are the two most likely causes of failed extractions?
Written-out numbers, ambiguous issue_type categories, customer_id not stated explicitly in the message.
