Lab D3-01

Claude Code Configuration

Domain 3 — Claude Code Task Statements 3.1, 3.2, 3.3 ⏱ 45 minutes

Objective: Configure Claude Code for a Python project using CLAUDE.md, settings.json, and MCP server wiring. Understand the permission hierarchy, hook configuration, and when Claude Code operates autonomously vs. when it pauses for human input.

Domain 3 is 20% of the exam. The CCAF tests not just "what is CLAUDE.md" but which configuration layer controls which behavior, and which settings cannot be overridden by the other layers.

Part 1

CLAUDE.md — Project Context File (15 min)

The CLAUDE.md file lives at the project root and gives Claude Code persistent context about the project — tech stack, conventions, what to do and not do. It's the equivalent of a new developer's onboarding document, but read automatically at session start.

project-root/
  CLAUDE.md   # Project-level context — Claude reads this first
  .claude/
    settings.json   # Tool permissions, MCP servers, hooks
    commands/
      deploy.md   # Custom slash command: /deploy

Write a CLAUDE.md for the Customer Support Agent Project

# CLAUDE.md for claude_architect project
# File: C:\Users\tsummey\projects\claude\claude_architect\CLAUDE.md

"""
# Customer Support Agent — Claude Code Context

## Project Overview
This is a Claude-powered customer support resolution agent. The primary
metric is First Contact Resolution (FCR) ≥ 80%.

## Tech Stack
- Python 3.11+ with anthropic SDK
- Simulation: scenario_training/*/simulation/agent.py
- Test runner: test_runner.py with --dry-run and --case flags

## Code Conventions
- Tool results MUST include: isError (bool), errorCategory, isRetryable (bool)
- SessionState must be a dataclass — never reconstruct from conversation history
- PreToolUse hooks block BEFORE tool execution — no exception
- All monetary values: float, USD, 2 decimal places

## Do Not
- Commit API keys or ANTHROPIC_API_KEY to git
- Use narrative termination ("I'm done") — only stop_reason drives loop exit
- Modify test_runner.py expected_outcomes without updating dataset.json

## Test Commands
  python test_runner.py --dry-run     # Preview all cases, no API calls
  python test_runner.py --case C011   # Test $749 refund case (hook must block)
"""
What CLAUDE.md Is NOT

CLAUDE.md is not a security boundary. It's context, not enforcement. Put business rules in hooks and gates — not in CLAUDE.md. Claude Code reads it as guidance, but it can't enforce business logic the way Python code can.

Part 2

settings.json — Permissions & MCP Wiring (20 min)

SettingPurposeValues
permissions.allowTools Claude Code can use without asking["Bash", "Read", "Edit"]
permissions.denyTools always blocked — cannot be overridden["WebFetch"] for air-gapped envs
mcpServersExternal tool servers available to ClaudeNamed server objects with command/args
hooksPre/post tool execution Python scriptsBash commands that run on tool events

Complete settings.json Configuration

{
  "permissions": {
    "allow": ["Read", "Edit", "Glob", "Grep", "Bash"],
    "deny": ["WebFetch"]
  },
  "mcpServers": {
    "customer-support-tools": {
      "command": "python",
      "args": ["mcp_server.py"],
      "env": {
        "DB_URL": "${CUSTOMER_DB_URL}"
      }
    }
  },
  "hooks": {
    "PreToolUse": [{
      "matcher": "Bash",
      "hooks": [{
        "type": "command",
        "command": "python .claude/hooks/pre_tool_check.py"
      }]
    }],
    "PostToolUse": [{
      "matcher": ".*",
      "hooks": [{
        "type": "command",
        "command": "python .claude/hooks/post_tool_log.py"
      }]
    }]
  }
}
Permission Hierarchy

deny always wins. If WebFetch is in the deny list, no CLAUDE.md instruction, no user message, and no hook can re-enable it during that session. This is the only enforcement layer Claude Code provides that is truly non-overridable at the prompt level.

Hook Script: Pre-Tool Bash Check

# .claude/hooks/pre_tool_check.py
# Claude Code calls this before every Bash tool use
# If script exits non-zero → tool call is blocked
import sys, json

tool_input = json.loads(sys.stdin.read())
command = tool_input.get("command", "")

BLOCKED_PATTERNS = ["rm -rf", "DROP TABLE", "git push --force"]
for pattern in BLOCKED_PATTERNS:
    if pattern in command:
        print(f"BLOCKED: '{pattern}' detected in command.", file=sys.stderr)
        sys.exit(1)  # Non-zero = block the tool

sys.exit(0)  # Zero = allow
Part 3

Autonomous vs. Pause-for-Input Decision Points (10 min)

Claude Code can run autonomously for low-risk, reversible tasks. It must pause for human confirmation on high-risk, irreversible, or ambiguous operations. Knowing this boundary is heavily tested.

ActionModeWhy
Reading files / searching codeAutonomousRead-only, fully reversible
Editing Python files in projectAutonomousIn permissions.allow, version controlled
Running test_runner.py --dry-runAutonomousExplicitly non-destructive flag
git push to main branchPauseIrreversible in production
Deleting filesPauseIrreversible without backups
Installing packages (pip install)PauseChanges environment — user should confirm
API calls with side effectsPauseExternal state changes cannot be undone
Exam Pattern

The exam will show a Claude Code scenario where it needs to make an irreversible change and ask: what should it do? The answer is always: pause and request explicit human confirmation, even if the task description implied autonomy. Autonomy ends at irreversibility.

Q

Reflection Questions

1. A developer writes in CLAUDE.md: "You may always run git push." Does this override the permissions.deny list?
Which layer has higher authority: CLAUDE.md context or settings.json permissions?
2. Your PreToolUse hook exits with code 1. What happens to the tool call Claude wanted to make?
Non-zero exit = blocked. Zero = allowed.
3. An MCP server crashes mid-session. What should Claude Code do? What should the MCP server's error response look like?
isError: true, errorCategory: "transient", isRetryable: true — and Claude should report the failure gracefully.
4. Where do you put the business rule "never delete production database records" — in CLAUDE.md, settings.json, or a hook? Why?
Which layer provides deterministic enforcement vs. model guidance?
5. Claude Code is asked to run a migration script that alters 50,000 rows. Should it proceed autonomously or pause?
Is this reversible? What class of action does this fall into?
